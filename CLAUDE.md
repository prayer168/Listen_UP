# 教師用語擴放站 — 開發者說明

## 專案概述

「教師用語擴放站」是一個部署於 GitHub Pages 的靜態網頁應用程式，
讓教師能一鍵全螢幕展示課堂用語，並自動語音朗讀。
資料儲存於 Firebase Firestore，身份驗證使用 Google 帳號。

## 管理員帳號

- Google 帳號：prayer168@gmail.com
- 此帳號擁有 Firebase 後端完整管理權限

## 技術決策

- **無框架**：純 HTML/CSS/JS，便於 GitHub Pages 直接部署，無需建置流程
- **Firebase Firestore**：即時資料庫，新增用語後前端立即更新
- **Web Speech API**：瀏覽器原生語音合成，支援繁體中文（`lang: 'zh-TW'`），無需第三方 API
- **縮放脈動動畫**：CSS `@keyframes` 實現，文字全螢幕顯示時持續縮放脈動

## 核心功能規格

### 全螢幕顯示
- 觸發：點擊左側用語列表項目，或在輸入框輸入後按下確定
- 效果：文字填滿整個視窗，`font-size` 動態計算以佔滿畫面
- 動畫：CSS `scale` 脈動（0.95 ↔ 1.05），週期約 1.5 秒
- 關閉：點擊任意位置或按下 `Escape`

### 語音朗讀
- 首次顯示時自動觸發 `SpeechSynthesisUtterance`
- 語言設定：`lang = 'zh-TW'`，語速：`rate = 0.9`
- 點擊全螢幕區域可重新朗讀

### Firebase 資料結構

```
firestore/
└── phrases/           # 集合
    └── {docId}/       # 每筆用語
        ├── text       # string  用語文字
        ├── category   # string  分類名稱（如「課堂秩序」）
        ├── order      # number  排序權重
        ├── createdAt  # timestamp
        └── createdBy  # string  建立者 email
```

### Firestore 安全規則（規劃）
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /phrases/{docId} {
      allow read: if true;                            // 所有人可讀
      allow write: if request.auth.token.email == 'prayer168@gmail.com';  // 僅管理員可寫
    }
  }
}
```

## 預設用語資料

### 一、上課前與進教室
1. 請在教室外排好隊，保持安靜。
2. 班長整隊，確認全班安靜後再進教室。
3. 最後進來的人請輕輕關門。
4. 進教室後，請拉開椅子，檢查桌面與抽屜。
5. 請把需要的課本、習作與文具準備好。
6. 請安靜預習或複習，等待老師開始上課。
7. 請坐姿端正，眼睛看前方。

### 二、課堂秩序與專心提醒
1. 請安靜。
2. 請看白板。
3. 全班起立。
4. 請把音量降下來。
5. 請回到自己的位置。
6. 請停止手邊動作，眼睛看老師。
7. 請專心聽講，認真做筆記。
8. 有問題請先舉手，等老師點到再發言。
9. 請尊重正在發言的同學。
10. 如果發生糾紛，請先舉手請老師處理，不要私下爭吵。
11. 如果班級過於吵鬧，請到外面重新整隊。

### 三、觀看影片與聆聽說明
1. 請認真看影片。
2. 答案就在影片中，請邊看邊思考。
3. 請不要對影片做無關評論。
4. 有問題先記下來，影片結束後再舉手發問。
5. 請把影片中的重點寫在筆記本或學習單上。
6. 影片結束後，請準備分享你看到的重點。

### 四、分組討論與合作學習
1. 請開始分組討論。
2. 討論時請輕聲細語。
3. 請每個人都要參與討論，不要只讓一個人完成。
4. 請先聽完同學的想法，再提出自己的意見。
5. 請組長協助分配任務。
6. 請記錄員把大家的想法整理下來。
7. 請準備派一位同學代表小組分享。
8. 討論時間還有兩分鐘，請開始整理結論。

### 五、實驗操作與安全提醒
1. 實驗中請分工合作，輕聲細語。
2. 請先聽完老師說明，再開始操作。
3. 未經允許，請勿任意碰觸實驗器材。
4. 請按照步驟操作，不要自行改變實驗方法。
5. 使用器材時請注意安全。
6. 觀察到的結果請確實記錄，不要用猜的。
7. 如果實驗有多餘時間，可以再做一次或兩次，並取平均。
8. 實驗還有兩分鐘結束，請準備收尾。
9. 實驗結束，請停止動作。
10. 請將實驗器材整理好並繳回。
11. 請確認桌面沒有水漬、垃圾或殘留材料。

### 六、作業、習作與檢討
1. 請用心完成作業。
2. 請字體工整，答案完整。
3. 請用直尺畫線。
4. 請勿抄襲。
5. 請勿只寫注音，能寫國字就寫國字。
6. 請確實訂正錯誤。
7. 請翻到今天要檢討的頁面。
8. 請拿出紅筆，準備訂正。
9. 共同檢討時，請邊聽邊修正。
10. 有不懂的地方，請立即舉手反應。

### 七、下課前整理與離開教室
1. 下課前，請先整理好自己的座位。
2. 請清理桌面、抽屜與地面。
3. 請把椅子推入並對齊。
4. 值日生請留下來完成最後整理。
5. 值日生工作：掃地、清理抽屜、擦拭桌面。
6. 掃具請平均分配，使用完要放回原位。
7. 老師檢查完，才能離開教室。
8. 請整理好座位後，再安靜離開。
9. 下課請有秩序離開教室。

### 八、鼓勵、提醒與收束語
1. 請給自己和同學一點掌聲。
2. 今天做得好的地方，請繼續保持。
3. 請想一想：這個活動讓你學到了什麼？
4. 這個實驗的收穫或心得是什麼？
5. 有沒有特別的發現或經驗？
6. 如果再做一次，你會怎麼改良？
7. 請把今天的重點整理成一句話。
8. 請帶著今天學到的觀察方法，回到生活中繼續發現。

## 開發注意事項

- 所有 Firebase config key 不得 hardcode 進 GitHub；使用環境變數或在 `js/firebase.js` 中留下佔位符並加入 `.gitignore`
- 語音朗讀在部分瀏覽器需要使用者互動才能啟動（autoplay policy），首次點擊後即正常
- `phrases.json` 作為 Firebase 初始 seed data，可用 Node.js 腳本批次匯入
